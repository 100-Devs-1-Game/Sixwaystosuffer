# Contributing to Six Ways to Suffer
Thank you for choosing to contribute to **Six Ways to Suffer**.  
This document outlines the technical workflow for making changes, improving systems, and submitting pull requests.

The tone of the game may be unsettling — but the development workflow should not be.
-------------------------------------------------------------------------------------------------------------------------------------

## Requirements

Before you begin, install:

- **Godot Engine 4.x** (this project was made and lastly run on godot 4.5.1 but works flawlessly on any other stable releases)
- **Git** (any version works properly including using web GITHUB to edit files and upload files)
-------------------------------------------------------------------------------------------------------------------------------------

## Project Setup

1. **Fork** this repository to your GitHub account.
2. **Clone** your fork locally:
   ```bash
   git clone https://github.com/YOUR_USERNAME/Sixwaystosuffer.git
Open Godot and select Import.
Choose the project.godot file inside the cloned folder, you should find it inside a similiar path to: "C:..\Sixwaystosuffer\src\project.godot"
Press Import, You are now ready to modify, debug, or enhance the project.
